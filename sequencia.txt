Fluxo no Admin (primeiro teste real)

Brands → Brand: crie 1 brand (ex.: smclab)

Brands → BrandAsset: (opcional) suba um INTRO e OUTRO mp4

Mediahub → SourceVideo: suba uma live (mp4)

Cuts → Cut: crie um corte (ex.: 00:00:10 até 00:01:10)

Jobs → Job: selecione os cortes, marque as redes (TikTok, Instagram Reels, etc.), intro/outro

Na lista de Jobs: marque o job → Action: Enfileirar

Saída final vai para:
storage\media\exports\job_<id>.mp4





subir o server
python manage.py runserver

subir o worker
celery -A config worker -l INFO -P solo

---

API (ver API.md)
1. POST /api/register/ - criar usuário
2. POST /api/auth/token/ - obter JWT
3. POST /api/sources/ - upload vídeo
4. POST /api/cuts/ - criar cortes (source + cuts[])
5. POST /api/jobs/ - criar job (cut_ids, target_platforms, ...)
6. POST /api/jobs/<id>/run/ - enfileirar
7. GET /api/jobs/<id>/ - status + output_url quando DONE
8. POST /api/scheduled-posts/ - agendar postagem
